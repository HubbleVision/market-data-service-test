### 问题 2: HK/US Stock Indicators 404 — 测试用 V1 路径，服务端用 V2 统一指标接口

**影响**: ~40 测试用例

测试文件调用模式:
```python
# test_hk_indicators.py / test_us_indicators.py
resp = tester.client.get(f"/api/v1/hkstock/{indicator}", params=params)
resp = tester.client.get(f"/api/v1/usstock/{indicator}", params=params)
```

服务端实际路由:
```
GET /api/v2/indicators/{indicator}?market=hk&symbol=00700.HK&interval=daily
GET /api/v2/indicators/{indicator}?market=us&symbol=AAPL&interval=daily
```

**修复方案**: 将 HK/US indicators 测试改为调用 `/api/v2/indicators/{indicator}`，增加 `market=hk` 或 `market=us` 参数。

**影响**: ~35 测试用例

| 测试测试路径 | 服务端实际路径 | 状态 |
|------------|-------------|------|
| `/api/v2/market/quote/{market}` | `/api/v2/market/quote` (query param: market=) | 路径结构不同 |
| `/api/v1/stock/klines` (symbol_normalize) | 需确认是否仍存在 | 404 |
| `/api/v1/stock/indicators` (symbol_normalize) | 需确认是否仍存在 | 404 |
| `/api/v2/{market}/brokerq` | 存在，但服务端 500 错误 | 服务端 bug |

**修复方案**:
- `test_market_quote.py`: 改路径为 `/api/v2/market/quote?market=xxx&codes=yyy`
- `test_symbol_normalize.py`: 确认 V1 统一股票接口是否仍存在，否则迁移到 V2
- `test_kline_adj_factor.py`: 同上

### 问题 4: V2 新增指标 — 测试配置中列了但服务端可能未全部注册

**影响**: indicators 模块部分 404

`config.py` 中 `V2_INDICATORS` 列了 53 个指标，但测试中 `test_new_indicators.py` 调用 `/api/v2/indicators/{indicator}` 时部分返回 404。

需要逐个验证服务端注册了哪些指标路由。服务端代码中注册的指标列表需要和 `V2_INDICATORS` 配置对比。

---

### 问题 5: Crypto Proxy 404

测试: `GET /api/v2/crypto/proxy/open-interest/list?baseCoin=BTC`
服务端: `GET /api/v2/crypto/proxy/{endpointPath}`

可能原因: 路径参数中 `/` 被当作路由分隔符，而非 endpointPath 的部分。需要确认 Chi router 对 `{endpointPath}` 的匹配模式 (是否包含 `/`)。

---

### 问题 6: Crypto News 参数不匹配

测试调用 `/api/v2/crypto/news/list` 时大量 422 错误（缺少必要参数），以及 500 错误（`coinank_news` 表不存在）。

- 422: 测试传递的参数名与服务端期望的不一致（如 `type` vs `type_`）
- 500: 服务端数据库表缺失，非测试问题

这是 CoinAnk 上游 API 的认证问题，非测试脚本问题。测试可以暂时标记为 skip 或容忍。

### Phase 4: 环境与配置
